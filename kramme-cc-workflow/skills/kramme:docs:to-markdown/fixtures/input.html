User Management API

This API provides endpoints for managing user accounts in the system. All requests require authentication via Bearer token in the Authorization header.

Endpoints

GET /api/users

Returns a paginated list of all users. Supports the following query parameters:

  • page — Page number (default: 1)
  • limit — Results per page (default: 20, max: 100)
  • role — Filter by role (admin, editor, viewer)
  • search — Full-text search on name and email

POST /api/users

Creates a new user account. Required fields in the request body:

Field Type Required Description
email string Yes Must be a valid email address
name string Yes Full name of the user
role string No One of: admin, editor, viewer (default: viewer)
password string Yes Minimum 8 characters, must include a number

Error Codes

The API uses standard HTTP status codes. Common error responses:

  • 400 — Invalid request body or missing required fields
  • 401 — Missing or invalid authentication token
  • 403 — Insufficient permissions for the requested action
  • 404 — User not found
  • 429 — Rate limit exceeded (max 100 requests per minute)
