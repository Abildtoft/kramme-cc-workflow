# Product Review Dimensions

System-wide review dimensions for whole-product review. These dimensions emphasize cross-flow patterns and overall product coherence rather than individual feature correctness.

## Navigation and IA Coherence

- Does the navigation structure make sense to a new user?
- Are related features grouped logically?
- Is the hierarchy flat enough to find features without deep drilling?
- Are breadcrumbs or context indicators present for deep pages?
- Do URLs/routes follow a consistent, predictable pattern?
- Is there a way to get back to the starting point from any page?

## Feature Discoverability

- Can a new user find the product's primary features within the first session?
- Are entry points visible and logically placed?
- Are there features that exist but are effectively hidden?
- Do feature names and labels match what users would search for?
- Are new or advanced features introduced progressively?

## Onboarding and First-Run

- What does a brand new user see on first login?
- Is there guidance or context to orient the user?
- Can the user accomplish their primary goal quickly?
- Are setup steps minimal and clearly explained?
- Is there value delivered before the user configures everything?

## Cross-Flow Consistency

- Do similar actions work the same way across different parts of the product?
- Are confirmation patterns consistent (modals vs inline vs toast)?
- Is save/cancel behavior predictable across forms?
- Do loading and error states follow the same patterns?
- Is terminology consistent across the product?

## Dead Ends and Abandoned Transitions

- Are there flows that lead to blank pages or "coming soon" states?
- Do all navigation items lead somewhere useful?
- Can the user always navigate backward from any state?
- Are there empty states that don't guide the user toward their next action?
- Do pagination or filtering dead ends provide helpful messaging?

## Repeated Friction Points

- Is the same frustration likely to happen in multiple places?
- Are there patterns of unnecessary clicks or confirmations?
- Do users need to re-enter information that the system already has?
- Are there recurring slow loads or unresponsive interactions?

## Trust and Safety Cues

- Are destructive actions clearly distinguished from routine ones?
- Is there confirmation before irreversible changes?
- Is the user's data clearly accounted for (saved, synced, backed up)?
- Are permissions and access boundaries communicated?
- Does the product communicate what happens next after important actions?

## Copy and Expectation Management

- Is terminology consistent across the product?
- Do button labels describe their action accurately?
- Are error messages helpful and specific?
- Do success messages confirm what happened?
- Is developer-facing jargon hidden from the user?
- Do empty states communicate what the user should do?
